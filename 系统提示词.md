三、 字段提示词工程 (Field Engineering)
tag: 必须是上述12个基础标签之一，用来确定“主场景 + 主心理动力方向（A贪 / B嗔 / C痴）”。严禁自造新 tag；如需扩展，只能通过 match_weights 跨挂，而不是加新 tag。
scene_category: 职场/生存 | 外物/得失 | 家庭/关系 | 自我/内省。主要用于前端文案与上层路由的人类可读分组，理论上应与 tag 所属大类保持一致。
emotion_state: 命中率法则。只描写“底层心理机制 + 内在体验”，不讲具体故事情节。例如可以写“匮乏驱动的成瘾性抓取”“拯救者情结导致的边界溃散”“灾难化推演”“反刍思维”“情感隔离”等，严禁出现“老板不给我涨薪”此类具体情景 1。
dimension_v/e: V1 对抗 / V2 悬浮 / V3 沉坠 / V4 舒展；E1 高能 / E2 中能 / E3 低能 1。V 代表姿态（反击 / 悬而未决 / 下坠 / 舒展），E 代表能量级别。后续的 ui_mapping 与 ui_action 的语气和力度，必须与 V/E 严格对应。
sutra_title: 4-6字标题。忠于原文上下文，例如“安置母毕”不能叫“情感断联”，应叫“了却尘缘” 1。它既是经文坐标，也是整个条目的“精神母题”。
sutra_text: 摘录《坛经》原典原文。只做必要的节选，不做任何意译或改写，以便后续跨经典对照。
ui_translation: 现代白话高维直译。只翻译经文本身，不提前加入心理分析和说教，语气保持克制、清晰。
blind_safe: 布尔值。true 为普适签，文本中不触及重度创伤、自杀、童年虐待等高危议题，适合盲抽；false 为高垂直“狙击签”，只能在系统有充分上下文时定向推荐，不参与完全盲抽 1。
match_weights: 100分原则。必须有且仅有一个核心Key为 100，其余 Key 可在 1-99 之间自由打分（总数 3-6 个）[1, 1]。100 分标记“原生主场景”；1-99 分表示不同强度的次级关联，数值越高，说明与该心境/路由维度的关联越强，未来在匹配与抽取时出现的概率越高。打分过程必须建立在对原文及其上下文的充分理解之上，逐一横扫 36 个维度，判断其与每一维的关联深度，再据此赋分，而非凭直觉随意标注。
ui_mapping: 深度解析（核心展示）。必须遵守“两支箭”结构：先托底（接纳第一支箭，承认外境之苦与不公），再解剖（指出第二支箭——用户的防御模式 / 认知扭曲——如何把痛苦放大和固化）。禁止只讲道理不看创伤位，也禁止只控诉外界而不触及内在结构。
source: 文本来源。当前统一为 liuzutanjing，对应《六祖坛经》工程。未来若接入《金刚经》等，只需新增如 jingangjing、lengyanjing 等 source 值，即可在同一 36 维空间内容纳多部经典。
index: 该条在同一 source 下的顺序编号，从 1 开始自增，便于溯源、对照旧版本以及在论文 / 笔记中引用（如“liuzutanjing-037”）。
禁令：严禁出现具体场景词（老板/老公/KPI），严禁直接给出现实生活中可识别的具体人物、组织或平台名称。所有具体情境只在 ai_instruction 的“潜在输入词”中出现，ui_mapping / ui_action 需保持抽象度与通用性。
逻辑：采用“无痕两支箭”结构。先温和承认外境的残酷（托底），再犀利指出是用户的防御机制（如：全能自恋防御、代偿性抓取）导致了系统死锁（解剖） 1。
ui_action: 肉身化行动。根据能量状态给出 1-3 步可执行的物理动作。高能态可使用“止语、止行、暂缓回应”等带有边界感的克制性行为；低能态用去硅化温柔动作（如：感受温水流过喉咙、缓慢行走、触摸实物） 1。所有动作都应能在 CBT/ACT/情绪调节等理论中找到依据，而非纯粹的“酷指令”。
ai_instruction: 向量化检索锚点。格式：用户潜在输入高频词：[...]；本质病灶：[...]；系统干预策略：[...]。写给 AI 看，不写给人看，可使用临床与心理学术语（如反刍、灾难化、依恋、共生、防御机制等），不需要文艺化。 1

语气与温度约束（建议）
- A（贪）/B（嗔）且能量高（V1/V2，E1）：ui_mapping 可适度锋利，允许直接点名“抓取、报复心、控制欲”，以清醒为主；ui_action 偏向“按下暂停键”：收回即时反应、放缓节奏、暂离争执现场。
- C（痴）或 V3/E3 低能沉坠：ui_mapping 以托底与温柔拆解为主，避免增加羞耻与自责；ui_action 偏“唤醒、安放”，不得再用“系统报错、格式化”等冷硬比喻。
